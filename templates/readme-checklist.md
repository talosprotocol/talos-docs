# README Standardization Checklist

## Structure
- [ ] **Title + Repo Role**: Does it clearly state its place in the stack?
- [ ] **Abstract**: Is it academic in tone?
- [ ] **Introduction**: Does it define the problem/solution?
- [ ] **System Architecture**:
    - [ ] Includes Mermaid diagram?
    - [ ] Diagram is scoped to this repo?
- [ ] **Technical Design**:
    - [ ] lists key modules?
    - [ ] lists contracts/formats?
- [ ] **Evaluation**:
    - [ ] Is it measurable? OR
    - [ ] explicit "N/A" pointing to another repo?
- [ ] **Usage**: Quickstart and workflows?
- [ ] **Operational Interface**:
    - [ ] `make test` mentioned?
    - [ ] `scripts/test.sh` mentioned?
- [ ] **Security Considerations**: Threat model summary?
- [ ] **References**:
    - [ ] Internal specs first?
    - [ ] Stable external links only?
    - [ ] Numbered list [1], [2]?

## Style
- [ ] No broken links
- [ ] No "placeholder" text
- [ ] Mermaid diagrams render correctly
- [ ] Code blocks specify language
